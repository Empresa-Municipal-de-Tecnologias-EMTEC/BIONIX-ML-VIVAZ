## 3) Treinos e runners (Detector, Identificador, DetectorLeve)
- Detector (modelo principal) — treinar do zero:
	$env:RESUME=''; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorModel.Runner/DetectorModel.Runner.csproj -- --mode train

- Retomar treino do Detector (usar checkpoint em PESOS/DETECTOR):
	$env:RESUME='1'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorModel.Runner/DetectorModel.Runner.csproj -- --mode train --resume

- Identificador — treinar:
	dotnet run --project BIONIX-ML-VIVAZ/src/IdentificadorModel.Runner/IdentificadorModel.Runner.csproj -- --mode train

- IdentificadorLeve — treinar do zero (PowerShell):
	# remove checkpoints antigos (se houver) e inicia treino do zero
	if (Test-Path PESOS\IDENTIFICADOR_LEVE) { Remove-Item -Recurse PESOS\IDENTIFICADOR_LEVE }
	$env:RESUME='0'; $env:INITIAL_LR='0.005'; $env:EPOCHS='1000'; $env:LOSS_THRESHOLD='0.0005'; $env:SUPPRESS_OUTPUTS='1'; dotnet run --project BIONIX-ML-VIVAZ/src/IdentificadorLeveModel.Runner/IdentificadorLeveModel.Runner.csproj -c Debug

- IdentificadorLeve — retomar treinamento a partir do checkpoint (PowerShell):
	$env:RESUME='1'; $env:INITIAL_LR='0.005'; $env:EPOCHS='1000'; $env:SUPPRESS_OUTPUTS='1'; dotnet run --project BIONIX-ML-VIVAZ/src/IdentificadorLeveModel.Runner/IdentificadorLeveModel.Runner.csproj -c Debug

- DetectorLeve (20×20 MLP) — treinar / retomar / opções comuns:
	Treinar (modo padrão):
		dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveModel.Runner/DetectorLeveModel.Runner.csproj -c Debug -- --mode train
	Retomar com checkpoint:
		$env:RESUME='1'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveModel.Runner/DetectorLeveModel.Runner.csproj -c Debug -- --mode train --resume
	Teste rápido (salva 1 imagem e sai):
		$env:QUICK_TEST_SAVE='1'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveModel.Runner/DetectorLeveModel.Runner.csproj -c Debug -- --mode train
	Ajustar épocas / parada por perda / suprimir outputs (exemplos):
		$env:EPOCHS='50'; $env:LOSS_THRESHOLD='0.5'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveModel.Runner/DetectorLeveModel.Runner.csproj -c Debug -- --mode train
		$env:SUPPRESS_OUTPUTS='1'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveModel.Runner/DetectorLeveModel.Runner.csproj -c Debug -- --mode train

DetectorLeve B (conv+pool + MLP) — comandos específicos
 - Treinar (continuação a partir de checkpoint):
	 $env:RESUME='1'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveBModel.Runner/DetectorLeveBModel.Runner.csproj -c Debug -- --resume --optimizer adam --epochs 1000 --max-samples 512

 - Treinar do zero (remover checkpoints e iniciar):
	 if (Test-Path PESOS\CLASSIFICADOR_DETECTOR_LEVE_B) { Remove-Item -Recurse PESOS\CLASSIFICADOR_DETECTOR_LEVE_B }
	 $env:RESUME='0'; $env:INITIAL_LR='1e-3'; $env:WEIGHT_DECAY='1e-4'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveBModel.Runner/DetectorLeveBModel.Runner.csproj -c Debug -- --optimizer adam --epochs 1000 --max-samples 512

 - Teste rápido / detect-test (gera crops em SAIDA):
	 $env:RESUME='1'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveBModel.Runner/DetectorLeveBModel.Runner.csproj -c Debug -- --detect-test --detect-samples 20 --resume

 - Comandos de continuação com LR maior (sweep sugerido):

	 $env:RESUME='1'; $env:INITIAL_LR='1e-3'; $env:WEIGHT_DECAY='1e-4'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveBModel.Runner/DetectorLeveBModel.Runner.csproj -c Debug -- --resume --optimizer adam --epochs 500 --max-samples 512
	 # INITIAL_LR = 5e-4
	 $env:RESUME='1'; $env:INITIAL_LR='5e-4'; $env:WEIGHT_DECAY='1e-5'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveBModel.Runner/DetectorLeveBModel.Runner.csproj -c Debug -- --resume --optimizer adam --epochs 500 --max-samples 512
	 # INITIAL_LR = 1e-4 (segurança)
	 $env:RESUME='1'; $env:INITIAL_LR='1e-4'; $env:WEIGHT_DECAY='1e-5'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveBModel.Runner/DetectorLeveBModel.Runner.csproj -c Debug -- --resume --optimizer adam --epochs 1000 --max-samples 512

	Sugestões de hiperparâmetros e comandos úteis (PowerShell)
	- Continuação com taxa de aprendizado maior (tente 1e-3):
		$env:RESUME='1'; $env:INITIAL_LR='1e-3'; $env:WEIGHT_DECAY='1e-4'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveModel.Runner/DetectorLeveModel.Runner.csproj -c Debug -- --resume --optimizer adam --epochs 1000 --max-samples 512

	- Sweep de taxas iniciais (experimente em ordem: 1e-3 → 5e-4 → 1e-4):
		# 1e-3
		$env:RESUME='1'; $env:INITIAL_LR='1e-3'; $env:WEIGHT_DECAY='1e-4'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveModel.Runner/DetectorLeveModel.Runner.csproj -c Debug -- --resume --optimizer adam --epochs 500 --max-samples 512
		# 5e-4
		$env:RESUME='1'; $env:INITIAL_LR='5e-4'; $env:WEIGHT_DECAY='1e-5'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveBModel.Runner/DetectorLeveModel.Runner.csproj -c Debug -- --resume --optimizer adam --epochs 500 --max-samples 512
		# 1e-4 (valor de segurança se 1e-3 divergir)
		$env:RESUME='1'; $env:INITIAL_LR='1e-4'; $env:WEIGHT_DECAY='1e-5'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveModel.Runner/DetectorLeveModel.Runner.csproj -c Debug -- --resume --optimizer adam --epochs 1000 --max-samples 512

	- Alternativa: treinar do zero com LR alto e decaimento (quando reiniciando):
		if (Test-Path PESOS\CLASSIFICADOR_DETECTOR_LEVE_B) { Remove-Item -Recurse PESOS\CLASSIFICADOR_DETECTOR_LEVE_B }
		$env:RESUME='0'; $env:INITIAL_LR='1e-3'; $env:WEIGHT_DECAY='1e-4'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveModel.Runner/DetectorLeveModel.Runner.csproj -c Debug -- --optimizer adam --epochs 1000 --max-samples 512

	- Detect-test (gera crops e logs em SAIDA) — exemplo rápido 20 amostras:
		$env:RESUME='1'; dotnet run --project BIONIX-ML-VIVAZ/src/DetectorLeveModel.Runner/DetectorLeveModel.Runner.csproj -c Debug -- --detect-test --detect-samples 20 --resume

	- Dicas rápidas:
		- Se observar instabilidade (loss explodindo), reduza `INITIAL_LR` para 5e-4 ou 1e-4.
		- Experimente `WEIGHT_DECAY` entre 1e-5 e 1e-3 (regulariza se houver overfitting).
		- Use `--max-samples` para limitar amostras por época e acelerar experimentos curtos.

## 5b) Docker
- Build + run API + Demo using docker-compose (from repository root `BIONIX-ML-VIVAZ`):

```powershell
docker compose up --build
```

- After compose up, services are available (defaults):
	- Demo: http://localhost:5000
	- API:  http://localhost:5001

Note: If you want to embed trained weights into the `Vivaz.WASM` assembly before building images, copy the weights folder into `src/Vivaz.WASM/PESOS/<FOLDER>` and then run `docker compose build` so the embedded resources are included in the build stage.

- Build / publicar Vivaz.WASM (biblioteca WASM/C#):
	dotnet build BIONIX-ML-VIVAZ/src/Vivaz.WASM/Vivaz.WASM.csproj -c Release
	(empacotar/embarcar pesos na artefato WASM ainda é um passo manual adicional)

## Novas instruções para WASM cliente e pesos via API

- Instalar workload WASM tools no .NET (necessário para publicar para browser-wasm):
	dotnet workload install wasm-tools

- Publicar artefatos WASM (exemplo genérico):
	dotnet publish BIONIX-ML-VIVAZ/src/Vivaz.WASM/Vivaz.WASM.csproj -c Release -r browser-wasm -o BIONIX-ML-VIVAZ/src/Vivaz.WASM/wasm_publish --no-self-contained

- Serviço de pesos no `Vivaz.Api`:
	- Copie as pastas de pesos para `src/Vivaz.Api/wwwroot/pesos/` e (opcional) compacte como `*.zip`.
	- Exemplo PowerShell:

```powershell
	Copy-Item -Recurse PESOS\CLASSIFICADOR_DETECTOR_LEVE_B src\Vivaz.Api\wwwroot\pesos\CLASSIFICADOR_DETECTOR_LEVE_B
	Compress-Archive -Path src\Vivaz.Api\wwwroot\pesos\CLASSIFICADOR_DETECTOR_LEVE_B -DestinationPath src\Vivaz.Api\wwwroot\pesos\CLASSIFICADOR_DETECTOR_LEVE_B.zip -Force
```

- Configure a variável de ambiente usada pelo WASM para localizar a API de pesos antes de servir a demo (exemplo PowerShell):

```powershell
$env:VIVAZ_API_URL='http://localhost:5000'
dotnet run --project BIONIX-ML-VIVAZ/src/Vivaz.Api/Vivaz.Api.csproj -c Debug --urls "http://localhost:5000"
```

O runtime cliente (quando compilado para WASM) tentará baixar `VIVAZ_API_URL/pesos/CLASSIFICADOR_DETECTOR_LEVE.zip` ou `.../IDENTIFICADOR_LEVE.zip` quando não encontrar pesos embutidos.

## WASM embedding (weights)
- To embed the DetectorLeve weights into the Vivaz.WASM assembly (recommended for offline WASM usage):
	1. Copy your weights folder `CLASSIFICADOR_DETECTOR_LEVE_B` into `src/Vivaz.WASM/PESOS/CLASSIFICADOR_DETECTOR_LEVE_B`.
	2. Build the project — files under `src/Vivaz.WASM/PESOS` are embedded into the assembly as resources:
		 dotnet build BIONIX-ML-VIVAZ/src/Vivaz.WASM/Vivaz.WASM.csproj -c Release
	3. The API endpoints `/api/face/wasm/detectjson` and `/api/face/wasm/detectcrop` will prefer embedded weights when available.

Notes:
- The demo site (http://localhost:5001) calls the WASM-backed API on http://localhost:5000 by default. Ensure both servers are running.
- WASM endpoints added to the API:
	- POST http://localhost:5000/api/face/wasm/detectjson  (form file: `file`)
	- POST http://localhost:5000/api/face/wasm/detectcrop  (form file: `file`) — returns PNG
